핵심쏙쏙 Git 2주차

<1> 이슈(Issue)

1. 프로젝트에서 해결해야하는 문제
2. 커밋 메시지에 이슈 포함 -> 내용 #이슈번호

<2> 브랜치(Branch)

1. 독립적으로 작업할 수 있는 공간
2. 체크아웃시 로컬 자료들도 같이 변경됨

<3> 병합(Merge)

1. 각자 작업했던 브랜치 합치는 것
2. 병합 이후 병합 시점에 의존하지 않고 커밋 시점에 맞게 정렬됨
3. 작업이 완료된 브랜치는 삭제

<4> Merge conflict

1. 중복되는 부분을 사용자가 수정할 수 있도록 일부러 충돌 일으킴

<5> 원격 repo, branch

1. 원격repo 이름/branch  예) origin/main
2. Tracking 하는 branch를 기준으로 commit 이루어짐
