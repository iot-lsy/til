핵심쏙쏙 Git 3주차

<1> PR

1. PR(Pull Request) : 병합(merge)해달라고 요청
2. 패치 : 원격repo 내용으로 로컬repo 업데이트
3. 수정은 로컬repo에서 하고 push

<2> Fork

1. Fork : 원본 소스코드를 복사해서 새로운 독립적인 소프트웨어로 개발하는 것

<3> Amend

가장 최신의 commit만 수정 가능, 강제로 push

<4> Revert, Reset

( 주의! ) 되돌리기 하는 경우 다른 사람에게도 영향을 미칠 수 있으므로 나만 작업하는 특정 branch 하나에만 적용한다.

1. Revert : commit을 남기고 되돌리는 것
2. Reset : commit했던 내용도 같이 삭제
3. Reset 모드 : Soft, Mixed, Hard
4. Mixed : 작업내역은 남겨두고 히스토리만 되돌린 것
5. Hard : 작업내역까지 모두 초기화

<5> Stash

1. 프로젝트의 변경사항을 임시적으로 보관해둘 때 사용
2. Commit 된 적 없는 새로운 파일은 stash하지 않아도 checkout 가능

<6> 좋은 git 커밋 메시지를 작성하기 위한 7가지 약속

1. 제목과 본문을 한 줄 띄워 분리하기
2. 제목은 영문 기준 50자 이내로
3. 제목끝에 . 금지
4. 제목은 명령조로
5. 본문은 영문 기준 72자마다 줄 바꾸기
6. 본문은 어떻게보다 무엇을, 왜에 맞춰 작성하기
7. (영어) 제목 첫번째 글자 대문자로


1. Commit 메시지, 단위로 작성 -> 적절하게 끊어야 버그를 찾아내기 수월함

<7> 코드리뷰

PR하는 단계에서 댓글을 달면서 코드리뷰를 함

1. 코드의 품질을 높일 수 있다.
2. 다른 사람의 눈으로 버그를 빠르게 발견할 수 있다.
3. 서로의 지식을 나누면서, 더 나은 방법을 찾아낼 수 있다.
	-> 팀의 코드 품질이 높아진다.

<8> gitignore 

 충돌을 피하거나 개인정보를 보호하기 위해 사용

<9> readme

프로젝트나 소프트웨어 사용할 때 먼저 읽어야하는 정보를 적어두는 파일

1. Readme.md는 서식이 적용될 수 있게 markdown이라는 형식의 파일로 작성한다.
