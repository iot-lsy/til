핵심쏙쏙 Git 1주차

<1>

1. 버전 관리
2. 작업 단위 나누기
3. 협업해서 하나의 프로그램 만드는데 유리

- Git -> 로컬 버전 관리
- GitHub -> git + 원격 저장소 = 협업

<2>

1. Git 버전관리 -> 파일 정보를 commit 이용해서 프로젝트 상태 저장하고 조회
2. Commit -> 현재 프로젝트 상태 저장하는 것

<3>

1. 프로그래밍할 때 파일/폴더명은 정보가 담기도록 만들기

<4>
1. 원격 repo -> 클라우드 , 로컬repo -> 내컴퓨터
2. 원격 repo에 자동으로 commit이 반영되지 않으며 하나의 원격 repo에 여러 로컬 repo 연결이 가능하다.
3. 로컬repo에서 원격repo로 작업 내역 올리는 것 -> Push
4. 원격repo에서 로컬repo로 작업 땡겨오는것 -> Pull
5. 비어있는 로컬repo에 원격repo 작업내용 복사해오기 -> Clone
